## What Happens When You Click "Start Now"

### The Hero Task CTA Journey

When a user clicks the **"Start now â†’"** button on the hero task card:

**Step 1: Navigation Decision (Instant)**

The app determines which interaction pattern to use based on task type:

```jsx
if (task.type === 'wizard') {
  // Multi-step guided flow
  navigate(`/tasks/${[task.id](http://task.id)}/wizard`);
}
else if (task.type === 'checklist') {
  // Checklist with sub-items
  navigate(`/tasks/${[task.id](http://task.id)}/checklist`);
}
else if (task.type === 'tool') {
  // Interactive tool (calculator, form builder)
  navigate(`/tools/${task.toolId}`);
}
else if (task.type === 'education') {
  // Educational content + action
  navigate(`/tasks/${[task.id](http://task.id)}/learn`);
}
else if (task.type === 'external') {
  // External action (e.g., visit government website)
  showModal('external-task', [task.id](http://task.id));
}
```

**Step 2: Page Transition**

- Smooth slide animation (300ms)
- Loading state if data fetch required
- Breadcrumb appears: Dashboard â†’ [Phase] â†’ [Task Name]

**Step 3: Task Screen Loads**

- User sees task-specific interface (see patterns below)
- Progress saved automatically
- Can exit anytime (back button, bottom nav)

### Loading States

- Gray placeholder boxes that shimmer
- Same layout as actual content
- Smooth fade-in when real content loads

**Spinners (500ms+):**

- Blue spinning circle
- Optional text: "Loading your data..."
- Fade in after 500ms (don't show immediately)

**Progress Bars (Known Duration):**

- File uploads
- Document generation
- Multi-step processing

- Special animation (e.g., 50% confidence = bigger celebration)

### Error Handling

**Validation Errors (Form):**

- Red border on invalid field

- Scroll to first error
- Disable submit until fixed

**API Errors:**

- Toast notification: "Something went wrong. Try again?"
- Retry button
- Contact support link (if persists)

**Offline Mode:**

- Yellow banner: "You're offline. Changes will sync when you're back online."

- Queue actions for when online
